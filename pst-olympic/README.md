## Abstract

Olympic pictograms are universal visual symbols designed to represent each sporting discipline without relying on text. First formally deployed at the Tokyo 1964 Olympic Games to guide international visitors, they quickly evolved into a global graphic language. Today, they blend utilitarian wayfinding with the artistic identity of the host nation. However, creating high-quality, scalable, reconfigurable, and portable technical and artistic designs has historically been a challenging task. Despite significant advancements, generating them remains a persistent challenge. Traditional methods of embedding static graphics result in large file sizes and reduce resolution, often obscuring fine or critical details. This new pst-olympic package published on [GitHub](https://github.com/manthanwar/pst-art/tree/main/pst-olympic) eliminates these issues by generating graphics natively in PostScript and LaTeX, guaranteeing exceptional quality, highly scalable and dynamically reconfigurable parametric graphics while keeping file sizes to a minimum.

## Introduction

Olympic pictograms are universal visual symbols designed to represent each sport and discipline at the Games. First widely introduced at the 1964 Tokyo Olympic Games, pictograms serve as a vital cross-cultural communication tool, enabling athletes and international spectators to easily navigate host cities and identify sporting disciplines. These graphic illustrations bridge language barriers, allowing a global audience to understand the events. Over the decades, these icons have evolved from simple human silhouettes into dynamic, culturally expressive sets that reflect the artistic heritage of the host nation.

Creating Olympic pictograms via computational design relies on the intersection of programmatic vector graphics and digital typesetting defined by mathematical formulas, loops, and coordinates. This methodology combines the native vector-drawing capabilities of PostScript with the macro-handling and structural layout strengths of LaTeX. By utilizing PostScript procedures, designers can mathematically define the geometries of athletic figures—translating human anatomy and equipment into scalable paths. When integrated into LaTeX documents via packages like PSTricks, these scripts generate highly precise, resolution-independent iconography using automated code ideal for official event documentation, signage, and academic publishing.

## Installation

### With TeX Live / MiKTeX

#### Automated Installation

Open your terminal or command prompt and run:

```bash
tlmgr install pst-olympic
```

#### Manual Installation

1. Download the files `pst-olympic.sty` and `pst-olympic.pdf`.
2. Move `pst-olympic.sty` to your local LaTeX directory:
   `texmf-local/tex/latex/pst-olympic/`
3. Update your TeX file database by running `texhash` (or `mktexlsr`).

### Using dtx and ins files

You can unpack the pst-olympic.sty from pst-olympic.dtx documented latex and pst-olympic.ins install script file by running:

```bash
latex pst-olympic.ins
```

## Usage

To use this package, add the following to the preamble of your document:

```latex
\documentclass{article}
\usepackage{pst-olympic}

\begin{document}

This is inline character A lit by \psLedDisplay[ledWidth=10]

\psset{unit=1mm,fillstyle=solid,linestyle=none}%
\psset{ledColorOn=yellow,ledWidth=8,ledDistance=0}%

\rput(0,-8mm){\psLedDisplay{Volt ~ : ~ 0123 V}}

\end{document}
```

## Compilation

PSTricks generates PostScript code, which standard pdflatex or lualatex compilers cannot process directly. You typically use the following workflow:

* The DVI -> PS -> PDF chain: Compile your document using the latex command, followed by dvips, and finally ps2pdf (e.g., latex file.tex && dvips file.dvi && ps2pdf file.ps).
